# F12 Eject Remap + Bluetooth Pairing Assistant Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Two related quality-of-life features for users running Apple Magic Keyboards on Windows:
1. **F12 / Eject remap** — Apple's F12 = Eject is dead under Windows. Let users repurpose it to a useful Windows shortcut (Calculator, Search, Apps menu, Mail, Volume Mute, or "do nothing"), via a dropdown in Settings.
2. **Bluetooth Pairing Assistant** — On first launch, detect whether any Apple keyboard is connected via HID. If none, surface a "Pair your Magic Keyboard" page that opens the Windows Bluetooth settings panel and watches for a new Apple keyboard, then forwards the user into the existing Detect flow.

Both features share a single Rust helper for **enumerating Apple HID keyboards** (`vendor_id == 0x05AC`).

**Architecture:**
- **Shared:** new Rust module `src-tauri/src/keyboard/hardware.rs` with one Tauri command `enumerate_apple_keyboards() -> Vec<AppleKeyboardInfo>` (shells out to `Get-PnpDevice` via PowerShell, mirrors the project's existing PowerShell-IPC pattern).
- **F12 remap:** new Rust module `src-tauri/src/keyboard/f12_remap.rs` with two commands `f12_remap_get() -> F12Action` and `f12_remap_set(action: F12Action)`. Writes a single entry into the Windows Scancode Map registry (key `HKLM\SYSTEM\CurrentControlSet\Control\Keyboard Layout`, value `Scancode Map`), composing it with any existing modifier-remap entries already managed by `scancode_map.rs`. The existing module already has read/write/clear primitives — the new module reuses them rather than duplicating registry logic.
- **F12 UI:** new section in `Settings.svelte` (alongside Cmd↔Win toggles in `Modifiers.svelte` is the natural cousin, but the dropdown is presentational and Settings is where global per-user toggles live in the project today).
- **Bluetooth Pairing:** new Svelte page `src/pages/BluetoothPairing.svelte` reachable from a new conditional branch on `Welcome` (when `enumerate_apple_keyboards()` returns empty AND `appState.skippedPairing !== true`). UI: "No Apple keyboard detected" → button "Open Bluetooth settings" → poll every 2s for max 5 min → on detection, auto-navigate to `Detect`.
- **Bluetooth watcher:** the polling lives entirely on the frontend (`setInterval` + `invoke("enumerate_apple_keyboards")`). No Rust-side event stream needed for MVP — keep it simple.

**Tech Stack:** Tauri v2, Svelte 5 runes, Rust (PowerShell shell-out for HID enumeration via `Get-PnpDevice`, registry I/O via the existing `scancode_map.rs` helpers), Vitest for frontend logic, `cargo test` for the F12 scancode-map composition.

**Out of MVP scope:**
- Persistent low-level keyboard hook for actions Windows doesn't expose as scancodes (Lock screen via Win+L, Snipping Tool via Win+Shift+S). Scancode Map only supports scancode → scancode remaps, not multi-key sequences. Defer to a future "MagicWindows tray helper" feature when code signing is in place.
- WMI event-driven pairing watcher (`__InstanceCreationEvent`). Polling at 2s intervals is good enough for MVP UX.
- Auto-pair via `BluetoothLEAdvertisementWatcher` — Windows requires explicit user consent in the Bluetooth panel for security, so no useful automation is possible. We just open the panel.

```rust
#[cfg(target_os = "windows")]
#[tauri::command]
pub fn enumerate_apple_keyboards() -> Result<Vec<AppleKeyboardInfo>, String> {
    use std::process::Command;

    // We mirror the diagnostics.rs / health_check.rs pattern: shell out to
    // PowerShell. Get-PnpDevice is the canonical Windows enumerator and is
    // available on every supported target (Win10 1607+).
    let script = r#"
$ErrorActionPreference = 'Stop'
$devices = Get-PnpDevice -Class Keyboard -PresentOnly -ErrorAction SilentlyContinue |
    Where-Object { ($_.HardwareID -join ';') -match 'VID_05AC' } |
    Select-Object FriendlyName, @{Name='HardwareID';Expression={($_.HardwareID -join ';')}}, Status
ConvertTo-Json -Depth 3 -Compress -InputObject @($devices)
"#;

    let output = Command::new("powershell")
        .args(["-ExecutionPolicy", "Bypass", "-NoProfile", "-Command", script])
        .output()
        .map_err(|e| format!("spawn powershell: {e}"))?;

    if !output.status.success() {
        return Err(format!(
            "enumerate failed: {}",
            String::from_utf8_lossy(&output.stderr)
        ));
    }

    let json = String::from_utf8_lossy(&output.stdout);
    let trimmed = json.trim();
    if trimmed.is_empty() || trimmed == "null" {
        return Ok(vec![]);
    }
    parse_pnp_output(trimmed)
}

#[cfg(not(target_os = "windows"))]
#[tauri::command]
pub fn enumerate_apple_keyboards() -> Result<Vec<AppleKeyboardInfo>, String> {
    Ok(vec![])
}

/// Pure parser for the JSON shape PowerShell emits. Extracted so it's
/// unit-testable without spawning PowerShell.
fn parse_pnp_output(json: &str) -> Result<Vec<AppleKeyboardInfo>, String> {
    let v: serde_json::Value = serde_json::from_str(json)
        .map_err(|e| format!("parse pnp output: {e}\nraw: {json}"))?;

    // ConvertTo-Json on a single object emits an object; on an array, an
    // array. Wrap a single object into a one-element array.
    let arr = match v {
        serde_json::Value::Array(a) => a,
        serde_json::Value::Object(_) => vec![v],
        _ => return Ok(vec![]),
    };

    let mut out = Vec::with_capacity(arr.len());
    for item in arr {
        out.push(AppleKeyboardInfo {
            friendly_name: item["FriendlyName"].as_str().unwrap_or("").to_string(),
            hardware_id: item["HardwareID"].as_str().unwrap_or("").to_string(),
            status: item["Status"].as_str().unwrap_or("").to_string(),
        });
    }
    Ok(out)
}
```

The F12 scancode under PS/2 is `0x58`. The Scancode Map remaps source-scancode → destination-scancode. Destinations we support:

| Action      | Dest scancode | Windows VK            |
|-------------|---------------|-----------------------|
| Default     | (no entry)    | F12 (VK_F12, 0x7B)    |
| Disabled    | `0x0000`      | nothing               |
| Calculator  | `0xE021`      | VK_LAUNCH_APP2        |
| Search      | `0xE065`      | VK_BROWSER_SEARCH     |
| Mail        | `0xE06C`      | VK_LAUNCH_MAIL        |
| AppsMenu    | `0xE05D`      | VK_APPS (right-click) |
| VolumeMute  | `0xE020`      | VK_VOLUME_MUTE        |

- [ ] **Step 2.1: Define the enum and pure helpers**

Create `src-tauri/src/keyboard/f12_remap.rs`:

```rust
//! F12 / Eject key remap.
//!
//! Apple Magic Keyboards expose F12 as Eject, which has no useful action
//! under Windows. We let users repurpose it via a single Scancode Map entry
//! that maps source scancode 0x58 (F12) to one of a curated set of
//! destination scancodes that Windows handles natively.
//!
//! Compose-friendly: writes/reads through the same registry key the
//! modifier-remap module already manages (`scancode_map.rs`), so the F12
//! entry coexists with Cmd↔Win swap entries without conflict.

use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "camelCase")]
pub enum F12Action {
    /// No remap — F12 stays F12 (no entry in the Scancode Map).
    Default,
    /// F12 produces nothing.
    Disabled,
    Calculator,
    Search,
    Mail,
    AppsMenu,
    VolumeMute,
}

impl F12Action {
    /// Destination scancode pair (4-hex-char little-endian) for the Scancode
    /// Map entry. Returns None when the action requires no entry (Default).
    pub fn dest_scancode(&self) -> Option<&'static str> {
        match self {
            F12Action::Default    => None,
            F12Action::Disabled   => Some("0000"),
            F12Action::Calculator => Some("21E0"), // 0xE021 little-endian
            F12Action::Search     => Some("65E0"),
            F12Action::Mail       => Some("6CE0"),
            F12Action::AppsMenu   => Some("5DE0"),
            F12Action::VolumeMute => Some("20E0"),
        }
    }

    /// Match a destination scancode pair back to an action. Inverse of
    /// `dest_scancode`, used to derive the current state from the registry.
    pub fn from_dest_scancode(s: &str) -> Option<F12Action> {
        match s {
            "0000" => Some(F12Action::Disabled),
            "21E0" | "21e0" => Some(F12Action::Calculator),
            "65E0" | "65e0" => Some(F12Action::Search),
            "6CE0" | "6ce0" => Some(F12Action::Mail),
            "5DE0" | "5de0" => Some(F12Action::AppsMenu),
            "20E0" | "20e0" => Some(F12Action::VolumeMute),
            _ => None,
        }
    }
}

/// Source scancode for F12: 0x58, little-endian "5800".
const F12_SCAN_LE: &str = "5800";
```

```rust
use crate::keyboard::scancode_map::{read_scancode_map, write_scancode_map, RawScancodePair};

/// Filter out any existing F12 entry. Returns the list with all
/// non-F12-source pairs preserved (so modifier remaps survive).
fn pairs_without_f12(pairs: Vec<RawScancodePair>) -> Vec<RawScancodePair> {
    pairs
        .into_iter()
        .filter(|p| p.old_code.to_lowercase() != F12_SCAN_LE)
        .collect()
}

#[tauri::command]
pub fn f12_remap_get() -> Result<F12Action, String> {
    let pairs = read_scancode_map().map_err(|e| format!("read scancode map: {e}"))?;
    for p in &pairs {
        if p.old_code.to_lowercase() == F12_SCAN_LE {
            if let Some(a) = F12Action::from_dest_scancode(&p.new_code) {
                return Ok(a);
            }
        }
    }
    Ok(F12Action::Default)
}

#[tauri::command]
pub fn f12_remap_set(action: F12Action) -> Result<(), String> {
    let pairs = read_scancode_map().map_err(|e| format!("read scancode map: {e}"))?;
    let mut new_pairs = pairs_without_f12(pairs);

    if let Some(dest) = action.dest_scancode() {
        new_pairs.push(RawScancodePair {
            new_code: dest.to_string(),
            old_code: F12_SCAN_LE.to_string(),
        });
    }

    write_scancode_map(&new_pairs).map_err(|e| format!("write scancode map: {e}"))
}
```

```svelte
      const kbds = await invoke<AppleKeyboardInfo[]>("enumerate_apple_keyboards");
      return kbds.length > 0;
    } catch {
      return false;
    }
  }

  async function startPolling() {
    polling = true;
    timedOut = false;
    detectedNow = false;
    timeoutId = window.setTimeout(() => {
      stopPolling();
      timedOut = true;
    }, TIMEOUT_MS);
    intervalId = window.setInterval(async () => {
      if (await probe()) {
        stopPolling();
        detectedNow = true;
        appState.appleKeyboardConnected = true;
        // Brief beat so user sees the success state, then advance.
        setTimeout(() => { appState.page = "detect"; }, 800);
      }
    }, POLL_MS);
  }

  async function openBluetoothSettings() {
    try {
      await open("ms-settings:bluetooth");
    } catch {
      // Some Tauri shell-plugin configs disallow ms-settings: — best effort.
    }
    if (!polling) startPolling();
  }

  function skip() {
    stopPolling();
    appState.skippedPairing = true;
    appState.page = "welcome";
  }

  onMount(() => {
    // Auto-start polling so the user doesn't have to click twice in the
    // happy path (open settings → pair → wait).
    startPolling();
  });

  onDestroy(() => stopPolling());
</script>
```

```ts
async function saveF12(next: F12Action) {
  f12Saving = true;
```
