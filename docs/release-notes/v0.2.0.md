# MagicWindows v0.2.0

## What's new | Nouveautés

### Portable builds are now first-class in every release

Previous releases only shipped the NSIS installer. Starting with 0.2.0, each release also includes a **self-contained portable ZIP** for both x64 and ARM64 — unzip anywhere, run `magicwindows.exe` in place, approve the UAC prompt for the layout install, done. Handy for trying the app without committing to an installer, or for carrying it on a USB stick between machines.

*Chaque release contient désormais un ZIP portable pour x64 et ARM64, en plus de l'installeur NSIS. Décompressez, lancez `magicwindows.exe`, acceptez l'UAC — aucun installeur requis.*

### README walkthrough

The GitHub README now shows the full install flow in screenshots — welcome, keyboard detection, preview, install progress, success, and the optional Mac-style modifier remap (Cmd ↔ Ctrl, Option ↔ Cmd, Caps Lock → Ctrl). First-time visitors can see exactly what they're getting before downloading anything.

*Le README affiche maintenant le parcours d'installation complet en captures d'écran, incluant le remappage optionnel des modificateurs macOS.*

## For contributors | Pour les contributeurs

- **New DX helper.** `scripts/Fix-ClaudeCodeShiftEnter.ps1` repairs a well-known Claude Code × VSCode keybinding quirk where `Shift+Enter` does nothing and plain `Enter` inserts the letter `m` in the terminal chat input. Unrelated to MagicWindows itself, but common enough among contributors that it's worth shipping a one-shot fix. See [`docs/CONTRIBUTING.md`](../CONTRIBUTING.md#dev-environment-tips).
- **Clippy passes under `-D warnings`.** Build script doc comments converted to inner, and one `&[x.clone()]` swapped for `std::slice::from_ref(&x)`. CI stays green.
