# MagicWindows v0.4.1

A bug-fix release for the Bluetooth pairing flow and layout (re)installation.

*Une mise à jour corrective pour l'appairage Bluetooth et la (ré)installation des
layouts.*

## Fixes | Corrections

### Bluetooth Apple keyboards are detected again

The pairing screen now recognises Bluetooth-connected Magic Keyboards. Detection
previously matched only Apple's USB vendor id (`05AC`); a Bluetooth keyboard
reports `004C`, so it was never found — which left the app stuck on the pairing
screen even with the keyboard connected.

*L'écran d'appairage reconnaît désormais les Magic Keyboard en Bluetooth. La
détection ne cherchait que l'identifiant USB d'Apple (`05AC`) ; un clavier
Bluetooth rapporte `004C` et n'était donc jamais trouvé — bloquant l'app sur
l'écran d'appairage même clavier connecté.*

### No more flashing PowerShell windows

Every PowerShell/cmd call now runs hidden (`CREATE_NO_WINDOW`). The pairing
screen polls on a timer, so console windows used to flash on screen a few times
per second; they're gone. An in-flight guard also prevents overlapping probes.

*Chaque appel PowerShell/cmd s'exécute désormais masqué. L'écran d'appairage
interroge le système en boucle, ce qui faisait clignoter des fenêtres de console
plusieurs fois par seconde ; c'est terminé. Une garde empêche aussi les sondes
de se chevaucher.*

### Reinstalling an active layout no longer fails

Re-installing a layout whose DLL is currently loaded used to fail with "the file
is in use by another process". The installer now skips the copy when the DLL is
identical, and otherwise swaps the loaded file aside before writing the new one.

*Réinstaller un layout dont le DLL est chargé échouait avec « le fichier est en
cours d'utilisation par un autre processus ». L'installeur saute désormais la
copie si le DLL est identique, sinon il écarte le fichier chargé avant d'écrire
le nouveau.*

### Working "Send bug report"

The bug-report button now copies the diagnostics to your clipboard and opens a
pre-filled GitHub issue, instead of a `mailto:` link that opened a blank browser
tab on machines without a mail client.

*Le bouton de rapport de bug copie désormais les diagnostics dans le
presse-papier et ouvre une issue GitHub pré-remplie, au lieu d'un lien `mailto:`
qui ouvrait un onglet vide sur les machines sans client mail.*
